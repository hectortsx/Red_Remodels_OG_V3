# Red Remodels — Project Guide for Claude

## What this project is
Static marketing website for **Red Remodels**, a Denver-area home remodeling company.
Hosted on **AWS S3 + CloudFront**. Contact form handled by an **AWS Lambda** function.
No build framework — plain HTML/CSS/JS. Blog posts are Markdown → compiled to HTML via `npm run build:blog`.

## Branching rules (ALWAYS follow these)
- **Never commit directly to `main`.**
- Create a feature branch before making any changes: `git checkout -b feature/description`
- Merge to `main` only when the work is complete and verified.
- Push to `main` and deploy together in the same step.

## Deploying
```bash
# 1. Sync files to S3
aws s3 sync public/ s3://www.redremodels.com --delete --profile red-remodels-deployer

# 2. Invalidate CloudFront cache (use specific paths, not /*)
aws cloudfront create-invalidation \
  --distribution-id EKTYEDHVURFWC \
  --paths "/path/changed" "/other/path" \
  --profile red-remodels-deployer
```
- S3 bucket: `s3://www.redremodels.com`
- CloudFront distribution ID: `EKTYEDHVURFWC`
- AWS profile: `red-remodels-deployer`

## Blog system
- Write posts as Markdown in `blog/posts/<slug>.md` with frontmatter (title, description, date, tags, image, imageAlt).
- Run `npm run build:blog` to compile → generates `public/blog/<slug>/index.html`, updates `public/blog/index.html`, and updates `public/sitemap.xml`.
- Never hand-edit the generated HTML in `public/blog/` — always edit the `.md` source and rebuild.
- YouTube embeds: use `<div class="video-embed"><iframe ...></iframe></div>` in the Markdown body.

## Nav dropdown
- Controlled by JS `.is-open` class — click to open, click outside or select item to close.
- No hover behavior (intentional — removed by design).
- Active page indicator: `aria-current="page"` on the current page's nav link → styled red underline via CSS.
- The nav HTML in `public/index.html` is the source of truth — `build-blog.js` extracts it automatically for blog pages.
- Service pages maintain their own nav copy and must be updated manually when nav changes.

## Service pages
All 8 service pages live in `public/services/<slug>/index.html`:
- kitchen-remodels, bathroom-remodeling, home-additions, custom-carpentry
- outdoor-living-spaces, basement-finishing, flooring-installation, garage-conversions-adus

Each follows the same structure: hero → highlights bar → scope grid → process steps → cost guide → CTA section.
Hero background image set via `.svc-hero--<slug>` CSS class in `services.css`.

## Lambda / contact form
- Handler: `lambda/contact-handler/index.mjs`
- Deploy: zip `index.mjs` + `package.json` → upload to Lambda `red-remodels-contact-handler`
- Spam filtering: keyword blocklist in `isSpam()` — add new keywords to `SPAM_EMAIL_KEYWORDS` or `SPAM_NAME_KEYWORDS`
- reCAPTCHA v3 is enabled (site key in the HTML, secret in Lambda env vars)

## CSS conventions
- CSS custom properties defined in `desktop.css` and `pages/desktop/home/style.css`
- Key colors: `--color-red: #c0392b`, `--color-navy: #031a36`
- Fonts: Montserrat (headings), Lato (body) via Google Fonts
- Footer logo: use `red-remodels-logo.png` at `width="160" height="60"` (not the old `284_Red_Remodels_LOGO_VR_04.png`)
- CTA section logo (`.svc-cta-logo`): keep `284_Red_Remodels_LOGO_VR_04.png` at 100×100 (square icon)

## SEO requirements
Every page must have:
- Unique `<title>` and `<meta name="description">`
- `<link rel="canonical">` (one only — no duplicates)
- `og:image`, `og:image:width`, `og:image:height`, `og:image:alt`
- `aria-current="page"` on the active nav link
